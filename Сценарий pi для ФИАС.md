```
Твоя задача — искать aoguid для населённых пунктов, используя подключенные MCP сервера fias-search и playwright.

MCP-сервер fias-search ищет aoguid по названию населенного пункта и идентификатору региона, а MCP-сервер playwright позволяет, как описано ниже, по географическим координатам получать название населенного пункта и региона, если их нет или они некорректны в изначальных представленных данных.

Итак, из исходного описания населённого пункта тебе нужно выделить:

- нормализованное название населённого пункта (НП)

- регион населённого пункта, по которому ты определишь идентификатор региона из списка ниже

Если не удалось определить aoguid населённого пункта и после поиска по координатам с последующим поиском в ФИАС, то нужно попробовать определить, не входит ли искомый населённый пункт в более крупный населённый пункт. Как теперь обработать все данные? Всех данных много, в один контекст их класть незачем и поэтому можно просто запускать в цикле пачки по, например, 5 населённых пунктов. При этом незачем по несколько раз метчить один и тот же населённый пункт, уберём дубликаты:
```

```
убери дубликаты из @cities.jsonl, ключ — locality и region, оставь только первую запись для каждого locality и region, сохрани результат в файл cities_deduplicated.jsonl
```

Отлично, избавились от кучи лишней работы!

И теперь скрипт для обработки пачками по 5 записей:

```python
import json
import os
import subprocess
from datetime import datetime
from itertools import batched


BATCH_SIZE = 5

with open("cities_deduplicated.jsonl", "r", encoding="utf-8") as f:
    for index, batch_lines in enumerate(batched(f, BATCH_SIZE), 1):
        batch = [json.loads(line) for line in batch_lines]

        results_file = f"results/{index:04d}.jsonl"
        if os.path.exists(results_file):
            print(f"Файл {index:04d}.jsonl уже есть, пропускаем")
            continue

        prompt = f"""
JSONL данные населённых пунктов:
{batch}

Для каждого населённого пункт найди запись в ФИАС.

Результаты сохрани results/{index:04d}.jsonl, формат каждой JSONL-строки должен быть строго таким:

{{"locality": "деревня Кириллово", "region": "Республика Башкортостан", "latitude": 54.79697036743164, "longitude": 56.3057861328125, "aoguid": "...", "notes": ""}}

Здесь два новых поля, которых нет в изначальных JSONL данных — aoguid и notes, значения старых полей менять нельзя. В поле notes можешь записать краткое примечение к этому конкретному поиску ФИАС идентификатора, если на него надо обратить внимание человеку - иначе запиши туда пустую строку.

После завершения обработки этого диапазона остановись.
        """.strip()
        ts = datetime.now().strftime("%H:%M:%S %d.%m.%Y")
        print(f"[{ts}] пакет {index}")
        subprocess.run(["pi", "--verbose",  "-p", prompt])
```

Запускаем и наблюдаем, как появляются результаты в директории `results`. Это небыстро, но зато вы можете заниматься другой работой, пока модель выполняет эту работу.

 Конечно, результаты надо будет потом проверить, как местами вручную, так и полностью автоматизированно, например, проверить, что выявленные aoguid существуют в БД, указывают на населённые пункты и тд. Из почти 3 тысяч записей у меня косячных было около десяти, не помню точно. Их легко поправили и всё.

Вот таким образом можно использовать большие языковые модели. Если вы когда-то занимались нормализацией и метчингом данных, то вы понимаете, какой это геморой — а сейчас это можно делать легко, причём в том числе на локальной совершенно бесплатной, ничем не ограниченной модели.
