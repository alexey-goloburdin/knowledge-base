Ошибка говорит, что порт 8964 уже занят другим процессом — скорее всего, предыдущий экземпляр Hiddify не завершился корректно.

**Быстрое решение:**

1. Открой **Диспетчер задач** (Ctrl+Shift+Esc)
2. Найди процессы связанные с Hiddify (или `sing-box`, `hiddify`)
3. Завершите их
4. Запусти Hiddify заново

**Если не помогло — проверь, что держит порт:**

Открой PowerShell от администратора и выполни:

```powershell
netstat -ano | findstr :8964
```

Увидишь PID процесса, который можно убить через:

```powershell
taskkill /PID <номер> /F
```

**Радикальный вариант:** просто перезагрузи компьютер — это гарантированно освободит порт.

---

Интересно — порт не занят, но bind всё равно падает. Это похоже на известную проблему Windows с **Hyper-V / WSL**, которые резервируют диапазоны портов.

**Проверь зарезервированные порты:**

```powershell
netsh interface ipv4 show excludedportrange protocol=tcp
```

Скорее всего увидишь, что 8964 попал в один из зарезервированных диапазонов.

**Решение — сменить порт в Hiddify:**

Настройки → Config Options → найди порт локального прокси и поменяй на что-то вне зарезервированных диапазонов (например, 10808 или 1080).

**Или радикально — отключить динамическое резервирование Hyper-V:**

```powershell
netsh int ipv4 set dynamic tcp start=49152 num=16384
```
