E2E-тесты, кто не в курсе, проверяют работу всего приложения в сборке. Если мы говорим о вебе, то E2E-тесты ведут себя как обычные пользователи. Открывают браузер, в нём нашу систему и протыкивают логику. Вызываются все слои: фронтенд, бэкенд, базы данных, очереди, всё. Можно протыкивать даже реальную систему, в которой работают пользователи. Ну, или развернуть идентичный реальному продакшну тестовый стенд.

С точки зрения выявления багов, то есть ошибок, это наиболее полезные тесты, потому что тестируют всё в сборке.

А потом мы просим модель создать детерминированный тест уже на языке программирования, например, на Python. Это просто песня. ## Старые наброски по теме

Тестовый проект лежит в `/home/sterx/code/tmp2`.

Run Playwright MCP server on Windows in powershell:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
npx @playwright/mcp@latest --port 3000 --host 0.0.0.0 --allowed-hosts '*'
```

Check IP:

```powershell
ipconfig
```

Get WSL address, for example, `172.27.112.1`.

`.mcp.json` for pi:

```json
{
  "mcpServers": {
    "playwright": {
      "url": "http://172.27.112.1:3000/mcp"
    }
  }
}
```

Run pi:

```shell
http_proxy="" https_proxy="" pi
```
